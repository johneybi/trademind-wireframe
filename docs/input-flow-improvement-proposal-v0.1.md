# TRADEMIND 입력 플로우 개선안 v0.1

> 목적: 현재 입력 화면이 자유 텍스트에 너무 많이 의존하는 문제를 줄이고,  
> AI 채팅 전에 필요한 최소 맥락을 더 안정적으로 확보하기 위한 개선안

> 작성일: 2026.04.04

---

## 1. 문제 정의

현재 입력 플로우는 마지막 `상황` 단계에서 자유 텍스트 하나에 많은 역할을 맡기고 있다.

사용자가 충분히 적지 않으면:

- 객관적 상황 정보가 부족해지고
- 사용자의 해석/심리 정보도 부족해지고
- AI 채팅이 시작된 뒤 질문 품질이 약해질 수 있다

즉, 지금 구조는 **사용자가 텍스트를 잘 써야만 서비스 품질이 유지되는 상태**에 가깝다.

---

## 2. 개선 방향

핵심 원칙:

- 스텝 수는 늘리지 않는다
- 순서만 조정한다
- 한 화면에 2종류 이상의 입력이 있더라도 의미가 가까우면 한 묶음으로 처리한다
- 자유 텍스트는 막연한 보완 입력이 아니라 `감정의 이유를 적는 필수 입력`으로 바꾼다

개선 방향 한 줄 요약:

> **종목 -> 상황 + 감정 -> 생각 + 이유 -> 브릿지**

즉 입력 단계는 이렇게 나뉜다.

- 상황을 먼저 짚고
- 그 상황에서 느끼는 감정을 고르고
- 마지막에 제일 마음에 걸리는 생각과 감정의 이유를 적는다

으로 묶는다.

---

## 3. 목표

이 개선안의 목표는 아래 세 가지다.

1. 사용자가 빈 텍스트칸 앞에서 막히지 않게 한다
2. AI가 최소한의 구조화된 맥락을 항상 받게 한다
3. 사용자가 `상황`과 `내 해석`을 자연스럽게 분리해 보게 한다

---

## 4. 현재 구조 vs 개선 구조

### 현재

1. 종목
2. 감정
3. 상황 자유입력
4. 브릿지

문제:

- `상황`과 `생각`이 한 텍스트칸 안에 섞인다
- 사용자가 아무 말이나 짧게 적으면 AI가 필요한 맥락이 부족하다
- 감정 직후 바로 긴 텍스트를 요구해서 부담이 크다

### 개선안

1. 종목
2. 상황 + 감정
3. 마음에 걸리는 생각 + 감정의 이유
4. 브릿지

장점:

- 사용자가 구조화된 입력부터 고를 수 있다
- 텍스트를 많이 못 써도 AI 채팅 품질이 덜 무너진다
- 사용자가 `사실`과 `해석`을 구분하게 된다
- `상황 -> 감정` 흐름이 자연스럽다
- 마지막 텍스트는 완전히 빈칸이 아니라, 이미 고른 상황/감정/생각을 바탕으로 적게 된다

---

## 5. 화면별 UI 개선안

### Step 1. 종목

목적:

- 어떤 종목 이야기인지 먼저 고정한다

UI:

- 검색형 인풋
- 자동완성 리스트
- 선택 후 종목 요약 카드

사용 컴포넌트:

- `Input`
- `Command`
- 선택 상태 카드

유지 원칙:

- 지금 구조를 크게 바꾸지 않는다
- 종목 입력 방식은 현재 흐름 유지

---

### Step 2. 감정

목적:

- 현재 어떤 상황에서 어떤 감정이 올라왔는지 먼저 묶어 잡는다

UI:

- 상황 선택 칩
- 단일 선택 감정 버튼 리스트

사용 컴포넌트:

- 칩 버튼 그룹
- 선택 버튼 리스트
- 라디오형 선택 표현

유지 원칙:

- 감정은 현재 단일 선택 유지
- Step 2는 `설명`보다 `빠른 분류`에 집중한다

#### 2-1. 객관적 상황

사용자에게 묻는 것:

> 지금 어떤 상황에 가까운가요?

UI:

- 단일 선택 칩

사전 모드 예시 칩:

- `급등 중이에요`
- `급락 중이에요`
- `횡보 중이에요`
- `손실 중이에요`
- `수익 중이에요`
- `뉴스를 봤어요`
- `주변 얘기를 들었어요`

사후 모드 예시 칩:

- `급등할 때 샀어요`
- `급락할 때 팔았어요`
- `손실 중이에요`
- `수익 중이에요`
- `뉴스 보고 들어갔어요`
- `주변 분위기에 휩쓸렸어요`

설계 의도:

- 감정 앞에 최소한의 상황 맥락을 먼저 고정한다
- 사용자가 "왜 이런 감정인지"를 말하기 쉽게 만든다

#### 2-2. 감정의 이유

사용자에게 묻는 것:

> 이런 상황에서 어떤 감정이 가장 크게 느껴지나요?

UI:

- 단일 선택 감정 버튼 리스트

권장 감정 옵션:

- `불안해요`
- `조급해요`
- `확신해요`
- `후회돼요`
- `모르겠어요`

설계 의도:

- 감정을 상황 바로 뒤에 붙여서 사용자가 끊기지 않게 한다
- 이 단계는 여전히 빠르게 지나가는 선택 단계여야 한다

---

### Step 3. 마음에 걸리는 생각 + 그 생각의 이유

목적:

- 선택한 상황과 감정을 바탕으로, 지금 제일 걸리는 생각과 그 이유를 한 번 더 분명히 하게 한다

이 단계는 두 블록으로 구성한다.

#### 3-1. 마음에 걸리는 생각

사용자에게 묻는 것:

> 지금 제일 마음에 걸리는 건 어떤 생각인가요?

UI:

- 단일 선택 칩

공통 예시 칩:

- `지금 안 하면 늦을 것 같아요`
- `더 떨어질까 불안해요`
- `본전 생각이 나요`
- `왜 그랬는지 후회돼요`
- `확신이 너무 커졌어요`
- `나만 놓칠까 조급해요`

설계 의도:

- 사용자가 현재 해석을 한 번 라벨링하게 한다
- 텍스트 입력 전에 생각을 먼저 고르게 해서 막막함을 줄인다

#### 3-2. 그 생각의 이유

사용자에게 묻는 것:

> 그 생각이 왜 계속 마음에 걸리나요?

UI:

- 2~4줄 높이 텍스트에리어
- 문장 시작을 돕는 시작 문구 버튼
- 선택한 상황, 감정, 생각을 요약해 보여주는 작은 안내문

권장 안내 문구:

- `길게 적지 않아도 괜찮아요. 지금 제일 걸리는 걸 적어주세요.`
- `왜 그 생각이 계속 남는지만 적어주세요.`

권장 시작 문구 버튼:

- `제가 제일 걱정하는 건...`
- `지금 놓치기 싫은 건...`
- `계속 머릿속에 도는 건...`
- `그때 제가 믿고 있었던 건...`

권장 플레이스홀더:

- `예: 지금 안 사면 영영 놓칠 것 같아서 계속 조급해져요`
- `예: 더 떨어질까 봐 머릿속에서 그 생각이 안 떠나요`
- `예: 왜 그때 들어갔는지 스스로도 납득이 안 돼서 계속 걸려요`

설계 의도:

- 텍스트는 여전히 필수지만, 완전히 빈칸에서 시작하지 않게 한다
- Step 2에서 잡은 `상황 + 감정`과 Step 3-1의 `생각`을 바탕으로 적게 한다
- 사용자는 마지막에 `왜 그 생각이 계속 남는지`를 자기 언어로 정리하게 된다

---

### Step 4. 브릿지

목적:

- 입력과 AI 채팅의 경계를 정리한다

UI:

- 짧은 안내 문구
- 단일 CTA

유지 원칙:

- 현재 방향 유지
- 뒤로/닫기 없이 단방향 전환 화면으로 둔다

---

## 6. 왜 스텝을 늘리지 않는가

스텝을 더 늘리면 얻는 것도 있지만, 지금 프로젝트에선 손해가 더 크다.

이유:

- 사용자가 입력 플로우를 더 길게 느낀다
- 해커톤 MVP 범위를 넘기 쉽다
- 0depth 이후 흐름이 너무 설문형으로 보일 수 있다

따라서 이번 개선의 핵심은

- 스텝 수 증가가 아니라
- `상황 단계 내부 구조화`

에 있다.

---

## 7. 권장 최종 구조

### 최종 스텝 순서

1. 종목
2. 상황 + 감정
3. 마음에 걸리는 생각 + 그 생각의 이유
4. 브릿지

### Step 2 권장 UI 순서

1. 객관적 상황 칩
2. 감정 선택

이 순서를 권장하는 이유:

- `상황 -> 감정` 흐름이 심리적으로 더 자연스럽기 때문이다

### Step 3 권장 UI 순서

1. 마음에 걸리는 생각 칩
2. `왜 이런 감정이 드셨나요?` 텍스트 입력

이 순서를 권장하는 이유:

- 텍스트 앞에 생각 칩을 두어, 사용자가 막막하지 않게 하기 위해서다

## 8. 구현 메모

프론트 구현 관점에서 필요한 상태:

- `situationTag`
- `thoughtTag`
- `detailText`

현재 `detail` 하나로 처리하던 부분을
최소한 위 3개로 나누어 관리하는 것이 적절하다.

다만 백엔드/LLM 계약은 바로 바꾸지 않아도 된다.

전달 직전에는 아래처럼 합칠 수 있다.

- `situationTag`
- `thoughtTag`
- `detailText`

를 프론트에서 조합해 하나의 `text`로 보낼 수 있다.

예:

```txt
상황: 급락 중이에요
생각: 더 떨어질까 불안해요
추가: 계속 손실이 커질까 걱정돼요.
```

즉, 화면 UX는 개선하되
기존 백엔드/LLM 계약을 당장 뒤엎지 않는 방향으로 갈 수 있다.

---

## 9. 현재 권장안

이번 개선안 기준 현재 가장 권장하는 방향은 아래와 같다.

- 스텝 수는 유지
- Step 2에서 `상황 + 감정`을 함께 받는다
- Step 3에서 `생각 + 그 생각의 이유`를 함께 받는다
- 텍스트는 필수 입력으로 보이게 설계한다
- 대신 완전히 빈칸처럼 느껴지지 않도록 생각 칩과 시작 문구를 먼저 제공한다
- 프론트 내부 상태는 나누고, 전송 직전에 기존 `text` 필드로 합침
