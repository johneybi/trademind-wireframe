# TRADEMIND UX Writing Guide

> 이 문서는 TRADEMIND의 모든 UI 문구 작성 기준입니다.
> 화면을 구현할 때 이 문서의 문구를 직접 사용하거나, 여기 정의된 원칙에 따라 작성하세요.

---

## 보이스 & 톤 원칙

### 핵심 포지션

TRADEMIND는 **옆에 앉아주는 동료**입니다.
조언하는 전문가도, 명령하는 시스템도 아닙니다.
사용자가 혼자 호가창 앞에 서 있을 때, 차분하게 곁에 있어주는 존재입니다.

### 4가지 원칙

| 원칙 | 설명 | 예시 |
|------|------|------|
| **공감 먼저** | 판단 전에 감정을 먼저 인정한다 | "불안한 건 당연해요" → 그 다음 질문 |
| **질문, 명령 아님** | 무엇을 하라고 지시하지 않는다 | "팔지 마세요" ❌ → "지금 파려는 이유가 뭔가요?" ✅ |
| **일상 언어** | 임상 용어를 UI에 그대로 쓰지 않는다 | "파국화" ❌ → "최악을 상상하고 있어요" ✅ |
| **짧게** | 감정이 격한 순간에 긴 문장은 읽히지 않는다 | 한 화면의 안내 문구는 2줄 이내 |

### 종결어미

- 기본: **`~해요`** (친근하지만 격식 유지)
- 질문: **`~인가요?`** / **`~할까요?`**
- 안내: **`~예요`** / **`~이에요`**
- 결과/완료: **`~했어요`** / **`~됐어요`**

### 타이포그래피 원칙

- 기본 UI는 `sans`
- 보조 설명과 AI 톤 문장은 제한적으로 `serif`

`serif`를 써도 되는 곳:

- 화면 부연설명 1~2줄
- 차분한 안내 문구
- AI의 공감 문장
- AI의 메타인지 질문

`serif`를 쓰지 않는 곳:

- 제목
- 버튼
- 탭
- 입력값
- 표, 차트, 통계
- 상태 태그

의도:

- `sans`는 구조와 행동을 담당
- `serif`는 잠깐 멈추고 생각하게 하는 문장에만 사용

즉, 제목을 꾸미기 위한 serif가 아니라 `판단을 늦추는 문장`을 위한 serif로 사용한다.

### 피해야 할 표현

| 금지 | 이유 | 대체 |
|------|------|------|
| "틀렸어요" / "잘못된 판단이에요" | 사용자를 판단하는 언어 | 판단 없이 질문으로 대체 |
| "절대 하지 마세요" | 지시·명령 | "잠깐 멈춰볼까요?" |
| "정상입니다" / "괜찮습니다" | 병원식 어투 | "그럴 만해요" / "당연해요" |
| "CBT", "파국화", "인지 왜곡" | 임상 용어 | 아래 인지 왜곡 언어 매핑 참고 |
| "AI가 분석했습니다" | 차갑고 기계적 | "살펴봤어요" / "보이는 게 있어요" |
| "입력하세요" | 명령형 | "적어주세요" / "말해주세요" |
| "오류가 발생했습니다" | 차갑고 무책임 | "잠깐 연결이 안 됐어요. 다시 시도해볼까요?" |

## 인지 왜곡 — UI 표시 언어 매핑

임상 용어는 내부 데이터용으로만 사용하고, 사용자에게는 아래 일상 언어로 표시합니다.

| 내부 용어 (distortion_type) | UI 태그 문구 | 부연 설명 (툴팁·채팅 내 설명용) |
|-----------------------------|--------------|----------------------------------|
| `catastrophizing` | 최악을 상상하고 있어요 | 아직 일어나지 않은 일을 이미 일어난 것처럼 느끼는 상태예요 |
| `all_or_nothing` | 지금 아니면 안 된다는 생각 | 중간이 없어요. 지금 이 순간이 전부처럼 느껴지는 거예요 |
| `emotional_reasoning` | 불안함이 신호처럼 느껴져요 | 감정이 사실처럼 느껴지는 순간이에요 |
| `confirmation_bias` | 믿고 싶은 것만 보이는 상태 | 확신이 강할수록 반대 신호가 잘 안 보여요 |
| `fomo_herd` | 나만 뒤처지는 것 같은 느낌 | 다들 타고 있는데 나만 못 타는 것 같은 조급함이에요 |
| `illusion_of_control` | 내가 잘 보면 될 것 같은 느낌 | 시장은 내 의지대로 움직이지 않아요. 그 긴장감이에요 |
| `anchoring_bias` | 그 가격이 머릿속에 박혀 있어요 | 처음 봤던 숫자가 기준이 돼버린 상태예요 |
| `sunk_cost` | 여기까지 왔는데 억울한 마음 | 이미 쓴 돈이 앞으로의 결정을 붙잡고 있어요 |

## 화면별 UX 문구

### 화면 1 — 메인

**헤드라인 (슬로건)**
```
지금 그 충동, 잠깐 멈춰요
```

**서브 문구 (시장 감정 요약)**
```
지금 투자자 68%가 조급한 상태예요
```
> 숫자는 실시간 집계. 감정 공유 보드 데이터 요약.

**핵심 CTA 버튼**
```
살까요?        팔까요?
```
> 두 버튼은 동등한 크기와 위계로. "?" 포함 — 명령이 아닌 함께 묻는 느낌.

**익명 감정 미리보기 레이블**
```
지금 다른 투자자들은
```

**익명 피드 예시 문구**
```
오늘 에코프로 때문에 멘탈 나갈 뻔 했어요
삼성전자 3일 연속 하락인데 버텨보기로 했어요
NVDA 급등 보고 참았는데 잘한 것 같아요
```

**하단 진입점 레이블**
```
멘탈 캘린더        감정 공유 보드
```

---

### 화면 2 — 입력

**모드 선택 질문**
```
지금 어떤 이야기를 하고 싶으세요?
```

**모드 선택 버튼**
```
사고 싶거나 팔고 싶어요        이미 한 매매가 마음에 걸려요
```
> 첫 번째 버튼 = 사전 모드 진입 / 두 번째 버튼 = 사후 복기 모드 진입.

**종목 입력 레이블**
```
어떤 종목인가요?
```

**종목 입력 플레이스홀더**
```
종목명을 입력하거나 선택하세요
```

**종목 인식 불가 fallback 메시지**
```
해당 종목 데이터를 가져올 수 없지만,
지금 느끼시는 감정에 집중해서 대화를 이어가 볼까요?
```

**종목 없이 진행 시 메시지**
```
특정 종목을 언급하지 않으셨지만,
지금 느끼시는 감정에 대해 함께 살펴볼게요.
```

**감정 선택 레이블**
```
지금 어떤 감정인가요?  (복수 선택 가능)
```

**감정 버튼**
```
불안해요        조급해요        확신해요        후회돼요        모르겠어요
```

**자유 텍스트 레이블**
```
조금 더 적어주세요  (선택)
```

**자유 텍스트 플레이스홀더**
```
지금 상황을 자유롭게 적어주세요
```

**다음 버튼**
```
다음
```
> 모드 선택 + 감정 최소 1개 선택 시 활성화. 비활성 상태에서 클릭 시 별도 안내 없이 버튼만 비활성 처리.

### 화면 3 — AI 채팅

**상단 인지 왜곡 태그** *(첫 AI 응답 후 표시)*
```
[인지 왜곡 매핑 테이블의 UI 태그 문구 사용]
예: ⚡ 지금 아니면 안 된다는 생각
```
> 아이콘은 감정의 강도와 성격에 따라 선택. 텍스트는 위 매핑 테이블 기준.

**건너뛰기 버튼** *(exploratory 질문에만 노출)*
```
건너뛸게요
```

**건너뛰기 시스템 메시지**
```
이 질문은 건너뛰었어요
```

**사전 모드 — 최종 메타인지 버튼 (3턴 말미)**
```
논리적 판단이 바뀌어서요        감정 때문인 것 같아요
```

**입력창 플레이스홀더**
```
자유롭게 적어주세요
```

**입력창 전송 버튼 레이블**
```
보내기
```

### 화면 4-A — 결과 (사전 모드)

**상단 문구**
```
한 발 물러서 보니 어떠세요?
```

**인지 왜곡 태그** *(화면 3에서 이어짐)*
```
[동일 태그 유지]
```

**커뮤니티 데이터 문구**
```
비슷한 상황의 투자자 78%가 관망을 선택했어요
```
> 숫자는 같은 `distortion_type` 기준 실시간 집계.

**익명 한줄 이유 헤더**
```
다른 투자자들의 이야기
```

**최종 선택 버튼**
```
잠시 관망하겠습니다        그래도 진행하겠습니다
```
> 두 버튼 모두 중립적 크기와 색으로. '진행' 쪽을 억제하지 않는다 — 사용자의 선택을 존중.

**관망 선택 후 확인 메시지**
```
잠깐 멈추는 것, 쉬운 선택이 아니에요.
오늘의 기록을 멘탈 캘린더에 남겼어요.
```

**진행 선택 후 확인 메시지**
```
알겠어요. 오늘의 기록은 남겨뒀어요.
나중에 돌아보는 게 생각보다 도움이 돼요.
```

### 화면 4-B — 결과 (사후 복기 모드)

**상단 문구**
```
오늘의 기록이 다음번을 바꿔줄 거예요
```

**인지 왜곡 태그** *(동일 유지)*

**커뮤니티 데이터 문구**
```
비슷한 상황에서 사후 후회를 느낀 투자자 89%
```

**기록 저장 확인 메시지**
```
오늘 기록을 저장했어요.
다음에 비슷한 충동이 올 때 이 기록을 보여드릴게요.
```

**멘탈 캘린더 이동 버튼**
```
멘탈 캘린더 보러 가기
```

## AI 대화 문구 가이드

AI가 생성하는 문구는 LLM 프롬프트 설계 영역이지만,
다음 원칙이 지켜지는지 프롬프트 설계 시 체크하세요.

### 구조 (항상 이 순서)

```
1. 공감 — 지금 감정이 당연하다는 것을 먼저 인정
2. 라벨링 — 일상 언어로 패턴을 설명 (임상 용어 금지)
3. 소크라테스식 질문 — 스스로 발견하게 유도
4. 메타인지 질문 — 이유의 이유를 묻기
```

### 공감 문구 예시

```
3일 연속 하락이면 정말 불안하셨겠어요.
많이 조급하시겠어요.
이미 매수하셨군요. 지금 돌아보는 것 자체가 중요한 첫 걸음이에요.
피 같은 돈이 줄어드는 걸 보면 불안한 건 당연해요.
```

### 라벨링 문구 예시

```
지금 느끼시는 감정, '지금 안 사면 이 기회를 영원히 놓친다'는 느낌이죠?
이런 순간을 '지금 아니면 안 된다는 생각'이라고 해요.
'이러다 전부 잃을 것 같다'는 느낌이 드시는 거죠?
이런 상태를 '최악을 상상하는 것'이라고 해요.
```

### 소크라테스식 질문 예시

```
15% 급등한 종목이 다음날도 추가 상승한 경우와 조정을 받은 경우, 어떤 쪽이 더 많을 것 같으세요?
'다들 산다'는 건 어디서 확인하셨나요?
처음 이 종목을 매수했을 때의 근거, 오늘도 여전히 유효한가요?
3일 연속 하락이 실제로 반토막까지 이어진 경우가 얼마나 될까요?
```

### 메타인지 유도 질문 예시

```
지금 매수하려는 이유가 '이 종목의 펀더멘털이 좋아서'인가요,
아니면 '나만 소외될 것 같은 불안감' 때문인가요?

지금 손절하려는 이유가 '매수 근거가 실제로 바뀌어서'인가요,
아니면 '빨간 숫자를 보는 게 너무 불안해서'인가요?

딱 하나만 고른다면요.
```

> 메타인지 질문 말미에 "딱 하나만 고른다면요."를 붙이면 사용자가 선택을 회피하지 않고 직면하게 됩니다.

### AI가 절대 하지 말아야 할 것

- 종목 추천 / 매수·매도 지시 / 수익률 예측
- "~하는 게 좋을 것 같아요" 형태의 방향 제시
- 임상 용어 직접 사용 (CBT, 파국화, 인지 왜곡, 메타인지)
- "틀렸어요" / "잘못된 선택이에요" 형태의 판단

## 상태 메시지 및 시스템 문구

### 로딩

```
잠깐요, 살펴보고 있어요
```

```
잠깐 연결이 안 됐어요.
다시 시도해볼까요?
```

```
답변을 가져오지 못했어요.
다시 보내볼까요?
```

```
멘탈 캘린더에 기록했어요
```

### 빈 상태 — 감정 공유 보드 피드

```
아직 오늘의 이야기가 없어요
```

### 빈 상태 — 멘탈 캘린더

아직 기록이 없어요.
첫 번째 쿨다운을 시작해볼까요?

## 숫자·통계 표기 원칙

커뮤니티 데이터와 감정 분포 수치는 다음 형식을 따릅니다.

```
비슷한 상황의 투자자 78%가 관망을 선택했어요
비슷한 상황에서 사후 후회를 느낀 투자자 89%
지금 투자자 68%가 조급한 상태예요
```

- "투자자의 78%가" ❌ → "투자자 78%가" ✅ (조사 '의' 생략)
- "약 78%" ❌ → 반올림 정수로 표기, '약' 생략
- 단위 없이 % 단독 표기

## 익명 피드 더미 데이터 가이드

MVP에서는 더미 데이터를 사용합니다.
아래 원칙에 따라 작성하세요.

- 종목명은 실제 한국/미국 주요 종목 사용 (삼성전자, SK하이닉스, 에코프로, NVDA, 테슬라 등)
- 구어체, 반말체 혼용 가능 — 실제 투자자가 쓸 법한 말
- 자책·판단 없이 담담하게 서술
- 결과를 단정하지 않음 ("잘했다" 단정 ❌ → "버텨보기로 했어요" ✅)

**예시 더미 문구**

```
오늘 에코프로 때문에 멘탈 나갈 뻔 했어요
삼성전자 3일 연속 하락인데 일단 버텨보기로 했어요
NVDA 급등 보고 30분 기다렸다가 안 샀어요
손절하고 나서 다음날 반등한 거 보고 멘탈 나간 적 있어요
급등 다음날 -8% 맞은 적 있어서 그냥 봤어요
FOMO인 거 알면서도 샀다가 후회한 게 한두 번이 아님
불안할 때일수록 차트 끄고 원칙 다시 읽어요
기록해두니까 패턴이 보이더라고요
나도 맨날 이러는데 기록해두니까 조금 달라졌어요
급등장에서 산 거 거의 다 후회함. 이제 30분 기다리는 규칙 만들었어요
```
