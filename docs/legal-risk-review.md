# TRADEMIND 법률 리스크 메모

작성일: 2026-04-01  
기준 범위: 대한민국 서비스 운영 전 검토용 초안  
주의: 이 문서는 내부 제품 기획용 메모이며, 법률 자문을 대체하지 않습니다. 출시 전에는 반드시 국내 변호사 또는 전문 자문을 통해 최종 확인이 필요합니다.

## 1. 한 줄 요약

TRADEMIND는 현재 기획 방향대로 `투자 추천 서비스`가 아니라 `투자 판단 점검 / 감정 기록 / 사후 복기 서비스`로 운영하면 법적 리스크를 줄일 수 있다. 다만 아래 두 축은 여전히 중요하다.

- `투자자문업 오인 리스크`
- `개인정보·민감정보 처리 리스크`

추가로 아래 이슈도 중요하다.

- `개인신용정보 확대 수집 리스크`
- `국외 이전 리스크`
- `익명 커뮤니티 재식별 리스크`
- `의료·심리치료 서비스 오인 리스크`

## 2. 우리 서비스에 가장 중요한 결론

### 2.1 해도 되는 방향

아래 방향은 상대적으로 안전하다.

- 사용자가 현재 감정과 판단 이유를 적는다
- AI가 공감, 질문, 메타인지 유도 중심으로 응답한다
- 결과 화면에서 비슷한 상황의 집계 데이터를 보여준다
- 사후 복기 기록을 통해 다음 의사결정 점검에 활용한다
- 서비스가 `무엇을 살까`가 아니라 `지금 사야 하는가`를 점검한다

### 2.2 위험해지는 방향

아래로 가면 투자자문 또는 규제 리스크가 커진다.

- `지금 사세요`, `지금 파세요` 같은 개별 실행 조언
- 특정 종목의 매수·매도 타이밍 제시
- 수익률, 목표가, 손절가 같은 구체적 숫자 제시
- 개별 사용자의 자산 상태에 맞춘 맞춤 투자 조언
- `추천`, `신호`, `고수익`, `수익 보장` 같은 카피

## 3. 주요 법률 리스크

### 3.1 투자자문업 리스크

자본시장법상 `투자자문업`은 금융투자상품의 가치 또는 투자판단에 관한 자문에 응하는 영업이다. 여기에는 종목, 취득·처분, 방법, 수량, 가격, 시기 판단이 포함된다.

TRADEMIND에 적용하면:

- `판단 점검`, `감정 기록`, `질문 유도` 중심이면 상대적으로 안전
- `개별 투자판단을 직접 제시`하면 위험
- 대화형 AI가 개인별 상황에 따라 사실상 매수·매도 결론을 주면 특히 위험

실무 해석:

- 서비스 설명, 버튼 문구, AI 답변, 결과 화면 전부에서 `투자 추천`으로 읽히지 않게 해야 한다
- 특히 채팅 UI는 사용자가 상담을 받는 느낌을 받기 쉬워서 더 조심해야 한다

### 3.2 개인정보 및 민감정보 리스크

감정 상태, 멘탈 패턴, 사후 복기 기록은 사용자를 식별할 수 있으면 개인정보다.

특히 이 서비스는:

- 감정 상태
- 심리 상태
- 반복 패턴
- 복기 기록

을 다루므로, 법문상 명시 여부와 별개로 `민감정보에 준하는 수준`으로 보수적으로 다루는 것이 안전하다. 실제 운영에서는 적어도 다음 수준이 필요하다.

- 별도 동의 검토
- 최소 수집
- 보유기간 명시
- 내부 접근권한 제한
- 전송/저장 보호조치
- 처리방침 명시

### 3.3 개인신용정보 리스크

현재 MVP처럼 `종목명 + 감정 + 자유 텍스트` 중심이면 주로 개인정보 리스크가 중심이다.

하지만 아래 데이터가 붙으면 신용정보법 이슈가 커진다.

- 실제 보유 종목
- 거래 내역
- 손익
- 계좌 정보
- 자산 규모
- 금융기관 연동 데이터

따라서 MVP 단계에서는 계좌 연동, 실시간 자산 연동, 세부 손익 추적은 되도록 피하는 것이 안전하다.

### 3.4 국외 이전 리스크

OpenAI API 등 해외 사업자 API를 쓰고, 사용자가 입력한 텍스트가 해외 서버로 전송되면 개인정보 국외 이전 이슈가 생길 수 있다.

실무상 확인해야 할 것:

- 어떤 데이터가 국외로 가는지
- 어느 국가/사업자로 가는지
- 처리 목적이 무엇인지
- 보유기간이 무엇인지
- 동의 또는 다른 적법 근거가 있는지
- 처리방침에 이를 적었는지

### 3.5 TOGETHER 보드의 재식별 리스크

`익명 한줄 사연`도 아래 정보가 합쳐지면 재식별 위험이 생긴다.

- 종목명
- 손실폭
- 날짜 또는 시간대
- 특정 사건
- 매우 특징적인 문장

따라서 공개용 텍스트는 원문 그대로 노출하지 않는 편이 안전하다.

권장 방식:

- 종목명 마스킹 또는 제거
- 손익 수치 제거
- 날짜/시점 일반화
- 원문 대신 요약/재작성
- 표본이 너무 적을 때는 텍스트 비노출

### 3.6 의료/심리치료 오인 리스크

이 서비스는 심리 상태를 다루지만, 의료 서비스처럼 보이면 별도 리스크가 생길 수 있다.

위험한 표현:

- `진단`
- `치료`
- `우울증 개선`
- `불안장애 완화`
- `정신건강 치료`

상대적으로 안전한 표현:

- `감정 기록`
- `판단 점검`
- `사후 복기`
- `쿨다운`
- `메타인지 유도`

## 4. 제품/디자인 가드레일

### 4.1 반드시 유지할 포지셔닝

- 이 서비스는 `종목 추천`을 하지 않는다
- 이 서비스는 `거래 실행`을 돕지 않는다
- 이 서비스는 `판단 점검`과 `감정 기록`을 돕는다
- 이 서비스는 `비슷한 상황의 투자자 데이터`를 참고하게 한다

### 4.2 UI/카피에서 피해야 할 표현

- 추천
- 매수 신호
- 매도 신호
- 최적 타이밍
- 수익 극대화
- 지금 들어가세요
- 손절하세요
- 이 종목은 오릅니다
- AI 종목 추천

### 4.3 UI/카피에서 권장되는 표현

- 판단 점검
- 감정 기록
- 타이핑 브레이크
- 사전 개입
- 사후 복기
- 관망
- 비슷한 상황의 투자자 선택
- 다음 질문
- 입력 요약

### 4.4 AI 응답 가드레일

AI는 다음처럼 답해야 한다.

- 공감으로 시작
- 질문 중심
- 선택을 강요하지 않음
- 종목 추천이나 가격 전망 금지
- 사용자의 논리와 감정을 분리해 보게 도움

금지 예시:

- `지금은 매도하는 게 맞습니다`
- `이 종목은 추가 상승 여력이 있습니다`
- `손실을 막으려면 바로 정리하세요`

허용 예시:

- `지금 판단의 근거가 바뀐 것인지, 불안이 커진 것인지 먼저 나눠볼까요?`
- `처음 이 종목을 봤을 때의 논리와 지금의 감정이 같은 방향인가요?`

## 5. MVP 운영 가이드

### 5.1 수집 최소화

MVP에서는 되도록 아래만 수집한다.

- 사용자 식별용 최소 정보
- 선택한 모드
- 감정 태그
- 자유 텍스트
- 최종 선택 결과
- 기록 생성 시점

되도록 수집하지 말 것:

- 주민등록번호
- 계좌번호
- 실거래 원장 수준 데이터
- 상세 자산 현황
- 과도한 행동 로그

### 5.2 동의 및 고지

출시 전 최소 필요 문서:

- 개인정보 처리방침
- 서비스 이용약관
- 민감정보 또는 이에 준하는 정보 처리 고지
- 국외 이전 고지 또는 동의 문구
- 커뮤니티 공개 범위 고지

### 5.3 보유기간

보유기간은 짧고 명확해야 한다.

예시 방향:

- 대화 원문: 목적 달성 후 단기 보관 또는 사용자 삭제 기능 제공
- 집계 데이터: 비식별·통계 목적 범위로만 유지
- 사후 복기 기록: 멘탈 캘린더 목적 범위에서만 유지

실제 기간은 법률 자문 후 확정 필요.

## 6. 출시 전 체크리스트

### 제품/카피

- [ ] `추천`, `신호`, `수익 보장` 표현이 없다
- [ ] AI가 개별 매수·매도 판단을 직접 제시하지 않는다
- [ ] 결과 화면 CTA가 투자 실행 유도가 아니라 판단 선택 중심이다
- [ ] 서비스 소개가 `멘탈 체크` 중심으로 되어 있다

### 데이터

- [ ] 수집 항목이 최소화되어 있다
- [ ] 감정·멘탈 데이터 처리 기준이 정리되어 있다
- [ ] 원문 텍스트 공개 여부와 익명화 방식이 정해져 있다
- [ ] 관리자 접근권한 정책이 있다

### 정책/법무

- [ ] 개인정보 처리방침 초안이 있다
- [ ] 국외 이전 고지가 정리되어 있다
- [ ] 커뮤니티 공개 고지가 있다
- [ ] 출시 전 외부 법률 검토를 받는다

## 7. 실무 권고

가장 현실적인 권고는 아래다.

1. MVP 단계에서는 `투자 추천` 요소를 완전히 배제한다
2. 계좌/자산 연동은 뒤로 미룬다
3. 감정·멘탈 데이터는 민감정보 수준으로 보수적으로 취급한다
4. 공개 커뮤니티 텍스트는 원문 공개보다 요약/재작성 방식을 우선한다
5. 실제 출시 전 변호사 리뷰를 받아 `투자자문`, `개인정보`, `국외 이전` 세 축을 확인한다
